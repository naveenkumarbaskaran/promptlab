# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please report them via one of these methods:

1. **GitHub Security Advisory** (preferred): Use the "Report a vulnerability" button on the [Security tab](../../security/advisories/new) of this repository.
2. **Email**: Send details to [naveenkb142@gmail.com](mailto:naveenkb142@gmail.com)

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 5 business days
- **Fix/patch**: Depends on severity, targeting within 30 days for critical issues

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| < Latest | ❌       |

## Security Best Practices

When using this package:

- Keep dependencies up to date
- Never commit API keys or secrets to version control
- Use environment variables for sensitive configuration
- Review dependency advisories regularly
